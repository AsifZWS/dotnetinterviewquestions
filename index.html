Programming Concepts FAQ (C#)

1. What are the different types of classes in programming?

Abstract Class:

  • Q: Can you create an object of an abstract class?
    A: No, you cannot create an object of an abstract class. You need to inherit from an abstract class to use it.
  • Q: What types of methods can an abstract class contain?
    A: An abstract class can contain both abstract and non-abstract methods. Abstract methods are declared with the abstract keyword, unlike interface methods.

Static Class:

  • Q: Can you create an object of a static class?
    A: No, you cannot create an object of a static class.
  • Q: Can a static class be inherited?
    A: No, a static class cannot be inherited.
  • Q: What can a static class contain?
    A: A static class can only contain static members (variables and methods), and these members must be declared with the static keyword.

Sealed Class:

  • Q: Can you create an object of a sealed class?
    A: Yes, you can create an object of a sealed class.
  • Q: Can you inherit from a sealed class?
    A: No, you cannot inherit from a sealed class.
  • Q: What members can a sealed class have?
    A: A sealed class can have normal members.

Partial Class:

  • Q: What does the partial keyword do for a class?
    A: The partial keyword allows a class to be split across multiple files. At runtime, all partial class files are merged into a single class. This is useful for collaborative work on a single class with different functionalities.

2. What are the different types of constructors in programming?

Default Constructor:

  • Q: What is a default constructor?
    A: A default constructor is a constructor without any parameters. It is automatically created if you do not define any constructors in the class.

Parameterized Constructor:

  • Q: What is a parameterized constructor?
    A: A parameterized constructor is one that takes parameters. You can have multiple parameterized constructors in a class, provided they have different parameter lists (constructor overloading).

Static Constructor:

  • Q: When is a static constructor called?
    A: A static constructor is called automatically when a class is first accessed. It executes only once.
  • Q: What are the restrictions for a static constructor?
    A: A static constructor cannot have access modifiers or parameters and can only access static members of the class.

Private Constructor:

  • Q: What is the purpose of a private constructor?
    A: A private constructor prevents the creation of objects from outside the class. It is often used in classes that contain only static members or in the Singleton design pattern.

Copy Constructor:

  • Q: What is a copy constructor?
    A: A copy constructor is a constructor that creates a new object by copying the variables from an existing object.
  • Example:
    Employee emp = new Employee();
    Employee emp1 = new Employee(emp);

3. What are extension methods in programming?

Extension Methods:

  • Q: What is an extension method?
    A: An extension method is a static method defined in a static class where the this modifier is applied to the first parameter. The type of this first parameter is the type being extended. For example, if you want to add a new method to the string type, you would create an extension method so that you can call it as if it were a native method like str.ToString().
  • Q: How do you use extension methods?
    A: Extension methods are only available when you explicitly import the namespace containing the static class with the extension methods into your source code using a using directive.

4. What is boxing and unboxing?

Boxing and Unboxing:

  • Q: What is boxing?
    A: Boxing is the process of converting a value type to a reference type.
    Example:
    int num2 = 123;
    object num1 = num2; // num1 is now a reference type holding the value of num2
  • Q: What is unboxing?
    A: Unboxing is the process of converting a reference type back to a value type.
    Example:
    object num1 = 123;
    int num2 = (int) num1; // num2 is now a value type holding the value of num1

5. What is Dependency Injection (DI) and how do scoped and transient dependencies differ?

Dependency Injection (DI):

  • Q: What is Dependency Injection?
    A: Dependency Injection (DI) is a design pattern used to achieve loose coupling between components by injecting dependencies rather than having components create them.
  • Q: What is a transient dependency?
    A: A transient dependency creates a new instance of the object each time it is requested. For example, if MyClass needs an instance of SomeDependency, it gets one, and if SomeOtherClass needs an instance, it gets a different, new instance of SomeDependency.
  • Q: What is a scoped dependency?
    A: A scoped dependency ensures that the same instance of the object is used throughout a single request. In the context of HTTP requests, if MyClass and SomeOtherClass both need an instance of SomeDependency, they will receive the same instance for the duration of that HTTP request.

6. What is middleware in ASP.NET?

Middleware in ASP.NET:

  • Q: What is middleware?
    A: Middleware is a pipeline component that sits between the HTTP request and response. It allows you to insert code for tasks such as authentication and logging in between the incoming request and outgoing response. Examples include app.UseRouting, app.UseAuthorization, and app.UseEndpoints.
  • Q: What is the purpose of app.Run() method in middleware?
    A: The app.Run() method adds a terminal middleware to the end of the pipeline. It typically does not call the next middleware component and is used to terminate the pipeline.
  • Q: How does app.Use() method differ from app.Run()?
    A: The app.Use() method configures middleware components and can call the next middleware in the pipeline. Unlike app.Run(), app.Use() can be used to chain multiple middleware components and can short-circuit the pipeline by not calling the next middleware.
  • Q: What is the app.Map() method used for?
    A: The app.Map() method is used to branch the request pipeline based on request path matches. It allows you to conditionally execute a branch of middleware if the request path starts with the specified path.
  • Q: What are the characteristics of middleware?
    A:
    • Middleware has access to both the incoming request and the outgoing response.
    • It can pass the request to the next piece of middleware in the pipeline.
    • It can perform processing logic before passing the request to the next middleware.
    • It can terminate (short-circuit) the request pipeline if needed.
    • Middleware is executed in the order it is added to the pipeline.

7. What is the difference between managed and unmanaged code?

Managed and Unmanaged Code:

  • Q: What is managed code?
    A: Managed code runs under the control of the Common Language Runtime (CLR) in the .NET environment. It benefits from features such as garbage collection and exception handling provided by the CLR.
  • Q: What is unmanaged code?
    A: Unmanaged code runs directly on the operating system without the CLR. It is typically compiled into machine code (e.g., executable files) and does not have the additional services provided by the CLR.

8. What are garbage collection, `IDisposable`, `Dispose`, and `Finalize` in .NET?

Garbage Collection and Disposal:

  • Q: What is garbage collection?
    A: Garbage collection in .NET is a process managed by the CLR that automatically frees up memory by deleting objects that are no longer in use. It helps prevent memory leaks.
  • Q: What is the `Finalize` method?
    A: The `Finalize` method is a special method in .NET that is called by the garbage collector to clean up unmanaged resources before an object is reclaimed. It is automatically invoked by the garbage collector and cannot be called explicitly.
  • Q: What is the `Dispose` method?
    A: The `Dispose` method is used to explicitly release unmanaged resources when they are no longer needed. To implement `Dispose`, a class should implement the `IDisposable` interface.
  • Q: What is the `IDisposable` interface?
    A: The `IDisposable` interface defines the `Dispose` method, which classes can implement to provide a way to release unmanaged resources manually.

9. What is the difference between `string` and `StringBuilder`, and when should you use each?

String vs StringBuilder:

  • Q: What is the `string` type?
    A: The `string` type is immutable, meaning its value cannot be changed once assigned. Every modification results in the creation of a new `string` instance.
  • Q: What is `StringBuilder`?
    A: `StringBuilder` is a mutable type designed for scenarios where the string value changes frequently. It is more efficient for such cases because it avoids creating multiple string instances.
  • Q: When should you use `StringBuilder` over `string`?
    A: Use `StringBuilder` when you expect to perform frequent modifications to a string value throughout the application lifecycle. It is more efficient in such cases compared to repeatedly modifying `string` instances.

10. What is the difference between `==` and `Equals` in object comparison?

Comparison Operators:

  • Q: What does `==` do in object comparison?
    A: The `==` operator compares object references, meaning it checks if two references point to the same object in memory.
  • Q: What does the `Equals` method do?
    A: The `Equals` method compares the actual values of the objects, assuming it is overridden in the object's class to provide value-based comparison.

11. What is the `Thread` class and its methods?

Thread Class:

  • Q: What is the `Thread` class used for?
    A: The `Thread` class is used to create and manage threads in .NET.
  • Q: What functionalities does the `Thread` class provide?
    A:
    • Creating foreground and background threads.
    • Setting the priority of a thread.
    • Getting the current state of a thread.
    • Providing a reference to the current executing thread.
  • Q: Can the `Thread` class be inherited?
    A: No, the `Thread` class is a sealed class and cannot be inherited.

12. What are `Task` and its functions in .NET?

Task Class:

  • Q: What is a `Task` in .NET?
    A: A `Task` represents an asynchronous operation and is part of the Task Parallel Library (TPL). It allows for easier and more efficient handling of asynchronous code compared to traditional threading.
  • Q: What functions does the `Task` class provide?
    A:
    • Creating and managing asynchronous operations.
    • Continuation tasks for chaining operations.
    • Handling results and exceptions of asynchronous operations.
    • Scheduling tasks to run concurrently.

13. What is late binding and early binding?

Late Binding vs Early Binding:

  • Q: What is late binding?
    A: Late binding refers to the process where the type of an object and its methods or properties are determined at runtime. It is used with dynamic objects where the type information is not known at compile time.
  • Q: What is early binding?
    A: Early binding refers to the process where the type of an object and its methods or properties are known at compile time. The compiler checks for type correctness before runtime.

14. What is the difference between `IQueryable` and `IEnumerable`?

IQueryable vs IEnumerable:

  • Q: What is `IQueryable`?
    A: `IQueryable` is best suited for querying data from a database. It allows for constructing queries that are translated to SQL and executed on the database server, providing efficient data retrieval.
  • Q: What is `IEnumerable`?
    A: `IEnumerable` is best for querying in-memory collections. It supports iteration over collections but does not provide query translation to SQL or other query optimizations.

15. What are the differences between `Array` and `ArrayList`?

Array vs ArrayList:

  • Q: What is an `Array`?
    A: An `Array` is a strongly-typed collection where all elements must be of the same data type. It is fast and efficient in terms of performance.
  • Q: What is an `ArrayList`?
    A: An `ArrayList` is a non-generic collection that can hold elements of different data types. It is slower and less efficient compared to `Array` due to its lack of type safety and overhead associated with boxing and unboxing.

16. How do you create a custom exception in .NET?

Creating a Custom Exception:

  • Q: What are the steps to create a custom exception?
    A:
    • Implement the `ApplicationException` class: Create a new class that derives from `ApplicationException`.
    • Define constructors: Provide constructors to initialize the exception with a message, inner exception, or both.
    • Override `Message` property: Optionally override the `Message` property to provide a custom error message.

17. What are the key concepts of Object-Oriented Programming (OOP)?

Fundamental Principles of OOP:

  • Q: What are the fundamental principles of Object-Oriented Programming (OOP)?
    A: The key principles of OOP are:
    • Encapsulation: Bundling data and methods that operate on the data within a single class.
    • Abstraction: Hiding complex implementation details and showing only the necessary features of an object.
    • Inheritance: Creating new classes that reuse, extend, and modify the behavior of existing classes.
    • Polymorphism: Allowing objects of different classes to be treated as objects of a common base class, with the ability to override methods for specific behaviors.

18. What is an interface, and what are its benefits and limitations?

Interface Details:

  • Q: What is an interface in programming?
    A: An interface is a contract that defines a set of abstract methods (without implementations) that implementing classes must provide. It is used to achieve abstraction and supports multiple inheritance.
  • Q: What are the benefits of using interfaces?
    A:
    • Abstraction: Interfaces provide a way to define methods without specifying how they are implemented.
    • Multiple Inheritance: A class can implement multiple interfaces, allowing for flexible designs.
    • Decoupling: Interfaces allow different parts of a program to interact without knowing the specific implementations.
  • Q: What are the limitations of using interfaces?
    A:
    • No Implementation: Interfaces cannot provide method implementations, so all implementing classes must provide their own.
    • No State: Interfaces cannot maintain state or hold data.
    • Overhead: Implementing multiple interfaces can increase complexity in the design and implementation of classes.

19. What do the keywords `virtual`, `ref`, `out`, and `yield` do in C#?

C# Keywords:

  • Q: What does the `virtual` keyword do?
    A: The `virtual` keyword allows a method in a base class to be overridden in derived classes. It indicates that the method can be redefined in any subclass.
  • Q: What is the purpose of the `ref` keyword?
    A: The `ref` keyword is used to pass arguments by reference to a method. Changes to the parameter in the method will reflect back to the original variable in the calling method. The `ref` parameter must be initialized before being passed.
  • Q: How does the `out` keyword differ from `ref`?
    A: The `out` keyword is used to pass arguments to methods as reference types, typically when a method needs to return multiple values. Unlike `ref`, `out` parameters do not need to be initialized before being passed.
  • Q: What does the `yield` keyword do?
    A: The `yield` keyword is used in an iterator method to return elements one at a time, allowing for deferred execution and lazy evaluation. It simplifies the implementation of custom iterators.

20. What is LINQ and what are its key methods?

LINQ Overview:

  • Q: What is LINQ?
    A: Language Integrated Query (LINQ) is a feature in .NET that allows querying of various data sources (such as in-memory collections, SQL databases, and XML) using a unified query syntax integrated into C#.
  • Q: What are some key LINQ methods and their purposes?
    A:
    • `Union`: Combines two sequences and returns unique elements from both.
    • `Intersect`: Returns only the elements that are present in both sequences.
    • `Lookup`: Groups elements by a key.
    • `Except`: Returns elements from the first sequence that are not present in the second sequence.

21. What are SOLID principles in software design?

SOLID Principles:

  • Q: What is the Single Responsibility Principle (SRP)?
    A: A class should have only one reason to change, meaning it should have only one responsibility.
  • Q: What is the Open/Closed Principle (OCP)?
    A: A class should be open for extension but closed for modification. This means you can add new functionality without changing existing code.
  • Q: What is the Liskov Substitution Principle (LSP)?
    A: Objects of a derived class should be replaceable with objects of the base class without affecting the correctness of the program.
  • Q: What is the Interface Segregation Principle (ISP)?
    A: A class should not be forced to implement interfaces it does not use. Instead, use multiple, smaller interfaces.
  • Q: What is the Dependency Inversion Principle (DIP)?
    A: High-level modules should not depend on low-level modules. Both should depend on abstractions. Abstractions should not depend on details. Details should depend on abstractions.

22. What is Entity Framework and the difference between Code First, Database First, and Model First approaches?

Entity Framework Overview:

  • Q: What is Entity Framework (EF)?
    A: Entity Framework (EF) is an Object-Relational Mapper (ORM) that allows developers to interact with a database using objects rather than SQL queries. It enables querying and manipulating data as strongly-typed objects.
  • Q: What is the Code First approach in EF?
    A: Code First allows developers to define the database schema using C# classes. The database is generated based on these classes.
  • Q: What is the Database First approach in EF?
    A: Database First generates C# classes based on an existing database schema. Developers work with these generated classes to interact with the database.
  • Q: What is the Model First approach in EF?
    A: Model First involves designing the data model using a visual designer, and then generating the database schema and C# classes from this model.

23. What are design patterns and why should we use them?

Design Patterns Overview:

  • Q: What are design patterns?
    A: Design patterns are proven solutions to common software design problems. They provide templates for solving specific issues in object-oriented design.
  • Q: Why should we use design patterns?
    A: Design patterns help improve code readability, reusability, and maintainability. They offer well-tested solutions and best practices that can be applied to various design problems.
  • Q: What is the Repository pattern?
    A: The Repository pattern provides an abstraction layer between the data access logic and the business logic. It allows for easy management and querying of data, and supports separation of concerns.
  • Q: What is the Unit Of Work pattern?
    A: The Unit Of Work pattern manages a set of operations that can be committed or rolled back as a single transaction. It helps ensure data consistency and manage transactional work.

24. What are collections, thread safety, and generics in .NET?

Collections, Thread Safety, and Generics:

  • Q: What are collections in .NET?
    A: Collections are data structures that store groups of related objects. Examples include arrays, lists, and dictionaries.
  • Q: What does thread safety mean in the context of collections?
    A: Thread safety ensures that a collection can be safely used by multiple threads simultaneously without causing data corruption or inconsistency.
  • Q: What are generics and why are they beneficial?
    A: Generics allow for the creation of classes, methods, and interfaces that operate on any data type while maintaining type safety. They are beneficial because they reduce code duplication, improve performance, and provide compile-time type checking.
  • Q: Which is faster: using generics or non-generic collections?
    A: Generics are generally faster than non-generic collections because they avoid the overhead of type casting and boxing/unboxing, leading to better performance and type safety.
